1. CONTENT SCHEMA — VERSION-CONTROLLED DESIGN
────────────────────────────────────────
Design a "Content" Mongoose schema with the following rules:

- Fields: title (String), genre (String), tags ([String]),
  language (String), authorId (ObjectId ref: User),
  contentType (enum: lyrics | story | poem | screenplay),
  quillDelta (Object),           ← current live version stored as Quill Delta JSON
  versions ([{                   ← embedded array, max 50 entries
    versionId: ObjectId,
    delta: Object,               ← Quill Delta snapshot, NOT full HTML
    editedAt: Date
  }]),
  bookmarkCount (Number, default 0),
  rating (Number, default 0),
  createdAt (Date), updatedAt (Date)

RULES:
- Embed up to 50 version snapshots inside the Content document.
- When the 51st version is added, pop the oldest snapshot out of the
  embedded array and write it to a separate "ArchivedVersions" collection
  ({ contentId, versionId, delta, editedAt }). This teaches the embed-vs-
  reference tradeoff explicitly.
- Never store full HTML — only Quill Delta diffs. A Delta is a compact JSON
  array of ops that is reconstructable to full content at any point.

ARCHIVING ROUTE:
POST /api/content/:id/versions
  - Append the new delta snapshot to versions[].
  - If versions.length > 50 before appending:
      • Shift the oldest entry out.
      • Insert it into ArchivedVersions collection.
  - Use a single findOneAndUpdate with $push + $slice: -50 to keep the
    array bounded atomically. Do NOT use two separate DB calls.

4. REDIS WRITE-THROUGH CACHE — BOOKMARK COUNTS
────────────────────────────────────────
Bookmark counts are read-heavy and must be served from Redis at all times.

POST /api/content/:id/bookmark   (authenticated)
  1. Update MongoDB: Content.findByIdAndUpdate(id, { $inc: { bookmarkCount: 1 } })
  2. Simultaneously (Promise.all): redis.incr(`bookmark:${id}`)
  → Both writes happen together — this is the write-through strategy.
  → Never queue the Redis write after the DB write resolves.

GET /api/content/:id  (public)
  - Check Redis: redis.get(`bookmark:${id}`)
  - If hit → return cached count directly (no MongoDB read for count).
  - If miss (first load) → read from MongoDB, populate Redis with
    redis.set(`bookmark:${id}`, count) with no TTL (counts are persistent).

UNBOOKMARK route:
POST /api/content/:id/unbookmark
  1. MongoDB: { $inc: { bookmarkCount: -1 } }
  2. redis.decr(`bookmark:${id}`)

IMPORTANT: Redis is the read source for bookmarkCount. MongoDB is the
source of truth for persistence. On server restart, seed Redis from MongoDB
for all active content: use a startup script that bulk-reads bookmarkCount
and sets all keys.

────────────────────────────────────────
Ratings are stored in a separate Ratings collection to avoid unbounded
document growth:
  { contentId: ObjectId, userId: ObjectId, score: Number (1-5), createdAt: Date }

    [ $match: { contentId: id } ]
    [ $group: { _id: null, avg: { $avg: "$score" }, count: { $sum: 1 } } ]

  - Redis: update cache key `rating:${id}` with the new average.

GET /api/content/:id/rating
  - Check redis.get(`rating:${id}`) first.
  - Miss → run the above aggregation + cache result with TTL: 300s.

## 2. ARCHIVING — MONGODB TRANSACTION (ATOMIC)

```js
// POST /api/content/:id/versions
const session = await mongoose.startSession();
session.startTransaction();
try {
  // Step 1: check if trim is needed before push
  const doc = await Content.findById(id).session(session);
  if (doc.versions.length >= 50) {
    const oldest = doc.versions[0];
    await ArchivedVersions.create([{
      contentId: id,
      versionId: oldest.versionId,
      delta: oldest.delta,
      editedAt: oldest.editedAt
    }], { session });
  }
  // Step 2: push new version, trim to last 50
  await Content.findByIdAndUpdate(id,
    { $push: { versions: { $each: [newSnapshot], $slice: -50 } } },
    { session }
  );
  await session.commitTransaction();
} catch (err) {
  await session.abortTransaction();
  throw err;
} finally {
  session.endSession();
}
```

Both the archive insert and the Content update succeed or fail together.
Version data cannot be lost due to a mid-operation failure.

## 5. REDIS — WRITE-THROUGH CACHE (BOOKMARK COUNTS)

```js
// POST /api/content/:id/bookmark  (auth required)
await Promise.all([
  Content.findByIdAndUpdate(id, { $inc: { bookmarkCount: 1 } }),
  redis.incr(`bookmark:${id}`)
]);
// Both writes fire simultaneously — this is write-through.

// POST /api/content/:id/unbookmark
await Promise.all([
  Content.findByIdAndUpdate(id, { $inc: { bookmarkCount: -1 } }),
  redis.decr(`bookmark:${id}`)
]);

// GET /api/content/:id  — Redis is primary fast-read layer; MongoDB is fallback
const cached = await redis.get(`bookmark:${id}`);
if (cached !== null) {
  logger.info(`[Redis HIT] bookmark:${id}`);
  bookmarkCount = parseInt(cached);
} else {
  logger.info(`[Redis MISS] bookmark:${id}`);
  const doc = await Content.findById(id, { bookmarkCount: 1 });
  bookmarkCount = doc.bookmarkCount;
  await redis.set(`bookmark:${id}`, bookmarkCount);  // lazy cache warm
}
```

> Cache seeding strategy: **lazy warming** (cache-aside on first read).
> Do NOT bulk-seed all content on startup — expensive at scale.
> Only hot/trending content may be optionally pre-seeded via a separate script.

## 7. RATING SYSTEM — RUNNING COUNTERS

```js
// POST /api/content/:id/rate  { score: 1-5 }
// Upsert rating; update running counters atomically
const existing = await Ratings.findOne({ contentId: id, userId });
if (existing) {
  const diff = score - existing.score;
  await Promise.all([
    Ratings.updateOne({ contentId: id, userId }, { $set: { score } }),
    Content.findByIdAndUpdate(id, { $inc: { ratingSum: diff } }),
    redis.set(`rating:${id}`, /* recomputed */ null)  // invalidate
  ]);
} else {
  await Promise.all([
    Ratings.create({ contentId: id, userId, score }),
    Content.findByIdAndUpdate(id, { $inc: { ratingSum: score, ratingCount: 1 } })
  ]);
}
// Invalidate rating cache — next read recomputes from counters
await redis.del(`rating:${id}`);

// GET /api/content/:id/rating
const cached = await redis.get(`rating:${id}`);
if (cached) return JSON.parse(cached);
const doc = await Content.findById(id, { ratingSum: 1, ratingCount: 1 });
const avg = doc.ratingCount > 0 ? doc.ratingSum / doc.ratingCount : 0;
await redis.set(`rating:${id}`, JSON.stringify({ avg, count: doc.ratingCount }), "EX", 300);
```

const validate = (schema) => (req, res, next) => {
  const result = schema.safeParse(req.body);

  req.body = result.data;
  next();
